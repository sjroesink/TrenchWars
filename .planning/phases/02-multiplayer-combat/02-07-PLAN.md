1. Create `packages/client/src/ship-select.ts`:
   - `ShipSelectOverlay` class:
     - Creates a simple HTML overlay div positioned over the game canvas
     - Shows: "Select your ship:" with three options:
       - "[1] Warbird - Fast, agile fighter"
       - "[2] Javelin - Powerful bomber"
       - "[3] Spider - Balanced all-rounder"
     - Listens for keypress 1, 2, or 3
     - Returns a Promise<number> that resolves with the ship type (0, 1, or 2)
     - After selection, removes the overlay
     - Style: semi-transparent black background, white text, centered, monospace font
     - Also accept mouse/click on the options

2. Modify `packages/client/src/main.ts`:
   - Before connecting to server, show ShipSelectOverlay and await selection
   - Pass selected shipType to sendJoin()
   - Store session token from WELCOME response (add sessionToken to WELCOME message)
   - On disconnect: attempt reconnect with session token after 2 seconds
   - On reconnect WELCOME: restore game state, clear prediction buffer

3. Modify `packages/client/src/network.ts`:
   - Add `sendReconnect(sessionToken: string)` method that sends a JOIN-like message with the token
   - Add auto-reconnect logic: on `ws.onclose`, wait 2 seconds, attempt to reconnect with stored session token (up to 3 attempts)
   - Add `onDisconnect` and `onReconnect` callbacks to handlers

4. Modify `packages/server/src/player-manager.ts`:
   - Add `sessionToken: string` field to PlayerState (generated via `crypto.randomUUID()`)
   - Add `disconnectedAt: number` field (tick when disconnected, 0 if connected)
   - Add `holdPlayer(id, tick)`: marks player as disconnected, sets disconnectedAt
   - Add `restorePlayer(sessionToken, tick)`: finds held player by token, if tick - disconnectedAt <= RECONNECT_TIMEOUT, marks as connected, returns PlayerState. Otherwise returns null.
   - Add `cleanupDisconnected(currentTick)`: removes players where currentTick - disconnectedAt > RECONNECT_TIMEOUT
   - A held player is NOT removed from the players map, is NOT included in alive players, but IS preserved for reconnection

5. Modify `packages/server/src/game-server.ts`:
   - On WELCOME response: include `sessionToken` from PlayerState
   - On disconnect: call holdPlayer instead of immediate removal
   - On JOIN message: check if it includes a sessionToken. If yes, try restorePlayer. If restored, send WELCOME with existing state. If not, create new player.
   - In tick(): call cleanupDisconnected(tickCount) periodically (every 100 ticks)

6. Write reconnect tests in `packages/server/src/__tests__/reconnect.test.ts`:
   - Test: holdPlayer marks player as disconnected
   - Test: restorePlayer with valid token within timeout returns PlayerState
   - Test: restorePlayer after timeout returns null
   - Test: cleanupDisconnected removes expired held players
