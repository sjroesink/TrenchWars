The 100Hz tick loop must complete each tick in < 10ms (1000ms / 100 ticks).
At 20Hz snapshot rate, broadcastSnapshot() runs every 5th tick.
Key bottleneck candidates:
1. JSON.stringify() per snapshot (called once, sent to N clients -- already efficient)
2. Physics loop: updateShipPhysics + applyWallCollision per player per tick
3. Weapon update: projectile movement + hit detection (O(players * projectiles))
4. Lag compensation recording: O(alive players) per tick
</interfaces>
