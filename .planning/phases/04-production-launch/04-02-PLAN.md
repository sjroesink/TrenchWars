  <action>
1. Create packages/server/src/arena-room.ts (class ArenaRoom):
   - Extract the per-room state from GameServer: PlayerManager, WeaponManager, LagCompensation, GameMode, tickCount, inputQueues map, clients map (WebSocket references for players in THIS room), disconnectTimers.
   - Constructor takes: { id: string, name: string, map: TileMap, maxPlayers?: number (default 20), gameMode?: GameMode }
   - Has its own tick() method (extracted from GameServer.tick()) that runs physics only for its players.
   - Has its own broadcastSnapshot() (extracted from GameServer).
   - Has addPlayer(playerId, name, shipType, ws) and removePlayer(playerId) methods.
   - Has broadcast() and send() helpers scoped to room clients.
   - Has getInfo(): { id, name, playerCount, maxPlayers, mode } for room listing.
   - Has start()/stop() for the room's game loop (setInterval, same BigInt accumulator pattern).

2. Create packages/server/src/room-manager.ts (class RoomManager):
   - Holds Map<string, ArenaRoom> of active rooms.
   - constructor(map: TileMap) creates 2 default rooms: "Arena 1" (FFA, 20 players), "Arena 2" (FFA, 20 players).
   - getRoomList(): Array<{ id, name, playerCount, maxPlayers, mode }> for sending to clients.
   - getRoom(id: string): ArenaRoom | undefined
   - assignPlayer(roomId, playerId, name, shipType, ws): boolean -- adds player to specified room. Returns false if room full.
   - removePlayer(playerId): void -- finds which room the player is in and removes them.
   - Has a playerRoomMap: Map<string, string> tracking playerId -> roomId for routing.
   - startAll()/stopAll() starts/stops all room game loops.

3. Refactor packages/server/src/game-server.ts:
   - GameServer becomes a thin WebSocket router. Remove playerManager, weaponManager, lagCompensation, gameMode, tick(), update() -- these now live in ArenaRoom.
   - Keep: wss setup, onConnection handler, message parsing.
   - Add RoomManager field. Create it in constructor.
   - New message flow:
     a. Client connects -> server waits for JOIN (same as before)
     b. Before JOIN, client can send new ROOM_LIST message -> server responds with ServerMsg.ROOM_LIST containing available rooms
     c. JOIN message gets new optional field `roomId?: string`. If omitted, assign to first non-full room.
     d. After JOIN, route INPUT/CHAT/SHIP_SELECT to the player's ArenaRoom via roomManager.
   - start() now calls roomManager.startAll() instead of its own loop.
   - stop() calls roomManager.stopAll().

4. Extend packages/shared/src/protocol.ts:
   - Add ClientMsg.ROOM_LIST = 0x06, ClientMsg.ROOM_JOIN = 0x07
   - Add ServerMsg.ROOM_LIST = 0x8b
   - Export these new values.

5. Update packages/server/src/main.ts:
   - No changes needed beyond ensuring GameServer still receives map (rooms are created internally).

IMPORTANT: The GameServer's existing onConnection/handleJoin/handleInput/handleChat methods should delegate to ArenaRoom instances. The WebSocket message parsing stays in GameServer, but the game logic lives in ArenaRoom.

Keep the existing single-room behavior working: if a client sends JOIN without roomId, auto-assign to the first room with space.
