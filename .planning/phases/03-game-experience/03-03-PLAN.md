    1. Create packages/client/src/ui/radar.ts -- Radar class:
       - Constructor takes `parent: Container` (the PixiJS stage) and `mapSize: number` (in tiles, typically 200 for the arena)
       - Creates a PixiJS Container positioned at bottom-right: `screenWidth - 160 - 16, screenHeight - 160 - 16`
       - Minimap dimensions: 160px x 160px
       - Background: draw a filled rect `0x000011` at alpha 0.7 in a Graphics object
       - Border: stroke rect `0xffffff` at alpha 0.2, width 1
       - Scale factor: `minimapPx / mapTileSize` (160 / mapSize). Store as `this.scale`
       - Player dots rendered in a separate Graphics object (cleared and redrawn each frame)
       - `updatePosition(screenWidth: number, screenHeight: number): void` -- repositions container when screen resizes. Position: `x = screenWidth - 160 - 16`, `y = screenHeight - 160 - 16`
       - `render(localPlayer: { x: number; y: number }, remotePlayers: Map<string, InterpolatedEntity>, screenWidth: number, screenHeight: number): void`:
         - Call updatePosition first
         - Clear dots graphics
         - Draw local player: circle at `(localPlayer.x * this.scale, localPlayer.y * this.scale)`, radius 3, fill `0x00ff88`
         - Draw remote players: for each alive entity, circle at `(entity.x * this.scale, entity.y * this.scale)`, radius 2, fill `0xffffff` (FFA mode -- all others white)
         - Do NOT render wall tiles on minimap for now (performance concern at 200x200; can add later as optimization)
       - `destroy(): void`

    2. Create packages/client/src/ui/chat.ts -- Chat class:
       - Constructor takes `onSend: (message: string) => void` callback, and `onActiveChange: (active: boolean) => void` callback
       - Creates container div `#chat-container` with `position: fixed; z-index: 200; pointer-events: none;`
       - Message history div: positioned above input area (bottom center, above 48px + 32px from bottom = ~96px from bottom), shows last 8 messages. Each message: `{name}: {message}` in 14px monospace `#ffffff`, name colored by ship type color (pass color as parameter). Background `rgba(0,0,0,0.5)` per message
       - Messages fade after 8000ms: set opacity to 0 with `transition: opacity 1s` after timeout
       - Hint text "Press Enter to chat" at bottom center, 48px from bottom, 14px monospace `#888888`, hidden when chat active
       - Input element: 400px wide, 32px tall, positioned at bottom center 48px from bottom. `background: rgba(0,0,0,0.8)`, `border: 1px solid rgba(255,255,255,0.3)`, `color: #ffffff`, 14px monospace. `display: none` initially. `placeholder: "Type a message..."`
       - `open(): void` -- show input, focus it, call `onActiveChange(true)`, hide hint
       - `close(): void` -- hide input, blur, call `onActiveChange(false)`, show hint
       - `send(): void` -- if input has non-empty trimmed value, call `onSend(value)`, clear input, close()
       - Keyboard handling ON THE INPUT ELEMENT (not window):
         - Enter: call send()
         - Escape: call close()
         - `e.stopPropagation()` on ALL keydown/keyup events on the input to prevent InputManager from receiving them
       - `addMessage(name: string, message: string, color?: string): void` -- adds message to history display, starts 8-second fade timer
       - Max 8 visible messages (remove oldest if > 8)
       - Window-level Enter key listener (when chat not active): calls open(). Must check `!this.active` to avoid re-opening
       - `destroy(): void`
