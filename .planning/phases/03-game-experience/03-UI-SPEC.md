# Phase 3 — UI Design Contract

> Visual and interaction contract for the game experience phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (game project — PixiJS canvas + programmatic HTML overlays) |
| Preset | not applicable |
| Component library | none (all UI is hand-built HTML/CSS positioned over PixiJS canvas) |
| Icon library | none (no icons needed; game uses geometric shapes and colored dots) |
| Font | `monospace` system stack (matches existing ship-select and debug panel) |

**Rationale:** This is a browser game, not a web app. All UI elements are absolutely-positioned HTML divs over a full-screen PixiJS canvas. No component library, CSS framework, or build-time CSS tooling is used or needed.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline text gaps, kill feed line spacing |
| sm | 8px | HUD element internal padding, radar border inset |
| md | 12px | Scoreboard row padding, chat message spacing |
| lg | 16px | HUD container padding, overlay section margins |
| xl | 24px | Scoreboard header padding, overlay title margin |
| 2xl | 32px | Major overlay section breaks |
| 3xl | 48px | Ship select vertical spacing (existing pattern) |

Exceptions: Radar minimap is 160px x 160px fixed size (not derived from spacing scale). Energy bar is 200px wide x 12px tall.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.4 | Chat messages, kill feed entries, scoreboard rows |
| Label | 12px | 600 (semibold) | 1.2 | HUD labels ("ENERGY", "K/D"), scoreboard headers, radar label |
| Heading | 22px | 600 (semibold) | 1.2 | Overlay titles ("SCOREBOARD"), round announcements |
| Display | 28px | 600 (semibold) | 1.0 | Ship select title (existing: 28px), game-over text |

All text uses `font-family: monospace` (system monospace stack). This matches the existing ship-select overlay and debug panel, and evokes the terminal/retro-tech aesthetic appropriate for an arena shooter.

## HUD Layout Contract

All HUD elements are absolutely-positioned HTML divs over the PixiJS canvas. Z-index layering:

| Layer | Z-Index | Contents |
|-------|---------|----------|
| Game canvas | 0 | PixiJS renderer (ships, projectiles, walls, radar) |
| HUD | 100 | Energy bar, K/D counter, kill feed |
| Chat | 200 | Chat input and message history |
| Scoreboard | 300 | Tab-toggled player list overlay |
| Full-screen overlay | 1000 | Ship select, game-over screen |

### Energy Bar (UIEX-02)

| Property | Value |
|----------|-------|
| Position | Bottom center, 16px from bottom edge |
| Size | 200px wide x 12px tall |
| Background | `rgba(255, 255, 255, 0.15)` (track) |
| Fill | CSS width% transition, color changes by threshold (see Game Colors) |
| Border | 1px solid `rgba(255, 255, 255, 0.3)` |
| Border radius | 2px |
| Animation | `transition: width 0.1s linear` (smooth drain/recharge) |
| Label | "ENERGY" in 12px semibold above bar, color `#888888` |

### Kill/Death Counter (UIEX-02)

| Property | Value |
|----------|-------|
| Position | Bottom-left, 16px from edges |
| Format | `K: {kills}  D: {deaths}` |
| Font | 14px monospace, weight 400 |
| Color | `#ffffff` |
| Background | `rgba(0, 0, 0, 0.5)` with 8px padding |

### Kill Feed (UIEX-02 extension)

| Property | Value |
|----------|-------|
| Position | Top-right, 16px from edges |
| Max entries | 5 visible at once |
| Entry lifetime | 5 seconds, then fade out over 0.5s |
| Format | `{killer_name} [weapon] {victim_name}` |
| Killer name color | `#00ff88` (accent) |
| Victim name color | `#ff3333` (destructive) |
| Weapon icon | Text: `>` for bullet, `*` for bomb |
| Font | 14px monospace, weight 400 |
| Background per entry | `rgba(0, 0, 0, 0.5)` with 4px vertical margin |
| Animation | Fade in: `opacity 0 -> 1` over 0.2s. Fade out: `opacity 1 -> 0` over 0.5s. |

### Radar / Minimap (UIEX-04)

| Property | Value |
|----------|-------|
| Implementation | PixiJS Container (NOT HTML) — rendered inside the game canvas |
| Position | Bottom-right corner, 16px inset from screen edges |
| Size | 160px x 160px |
| Background | `rgba(0, 0, 17, 0.7)` (dark space, semi-transparent) |
| Border | 1px solid `rgba(255, 255, 255, 0.2)` |
| Local player dot | 3px radius circle, `#00ff88` |
| Friendly player dot | 2px radius circle, `#4488ff` (or `#00ff88` in FFA) |
| Enemy player dot | 2px radius circle, `#ff4444` (or `#ffffff` in FFA for others) |
| Wall rendering | Scale map walls to minimap size, render as 1px dots in `#334466` at 0.3 opacity |
| Scale | `minimapSize / mapTileSize` = 160 / 200 = 0.8 px per tile |
| Update rate | Every frame (follows camera) |

### Scoreboard Overlay (UIEX-03)

| Property | Value |
|----------|-------|
| Trigger | Hold Tab key (show while held, hide on release) |
| Position | Centered on screen |
| Size | 480px wide, height auto (max 70vh with scroll) |
| Background | `rgba(0, 0, 0, 0.85)` |
| Border | 1px solid `rgba(255, 255, 255, 0.2)` |
| Title | "SCOREBOARD" in 22px semibold, centered, `#ffffff` |
| Columns (FFA) | Rank, Name, Ship, Kills, Deaths, K/D Ratio |
| Columns (Team) | Team header row, then: Name, Ship, Kills, Deaths, Lives |
| Row height | 32px |
| Row padding | 8px horizontal, 4px vertical |
| Local player row | Background highlight `rgba(0, 255, 136, 0.1)` |
| Sort order | FFA: by kills descending. Team: grouped by team, then kills descending |
| Header row | 12px semibold uppercase, color `#888888`, bottom border `rgba(255,255,255,0.1)` |
| Team headers | Full-width row with team color background at 20% opacity, team name in team color |

### Chat (UIEX-05)

| Property | Value |
|----------|-------|
| Toggle | Enter key opens input, Enter sends and closes, Escape cancels |
| Input position | Bottom center, above energy bar, 48px from bottom |
| Input size | 400px wide, 32px tall |
| Input style | `background: rgba(0,0,0,0.8)`, `border: 1px solid rgba(255,255,255,0.3)`, `color: #ffffff`, 14px monospace |
| Message history | Above input, last 8 messages visible |
| Message format | `{name}: {message}` |
| Name color | Ship color of sender |
| Message color | `#ffffff` |
| Fade | Messages fade after 8 seconds (opacity 0 over 1s) |
| Input capture | When chat input is focused, ALL keyboard events stop propagation to InputManager. Ship does not move while typing. |

### Ship Select Overlay (UIEX-01)

Already implemented in `ship-select.ts`. No changes needed to visual design. Integration change: audio initialization (Howler.js) should occur on the user's first interaction within this overlay (click or keypress) to satisfy browser autoplay policy.

### Game Over / Round End Screen

| Property | Value |
|----------|-------|
| Trigger | Server sends game-over or round-end event |
| Position | Centered overlay |
| Background | `rgba(0, 0, 0, 0.85)` |
| FFA winner text | "{name} wins!" in 28px semibold, color = winner's ship color |

| Round text | "Round {n} of {total}" in 14px, `#888888` |

## Visual Effects Contract (UIEX-07)

### Glow / Bloom

| Effect | Target | Library | Settings |
|--------|--------|---------|----------|
| Ship glow | Ship Container (all ships grouped) | pixi-filters GlowFilter | `distance: 10, outerStrength: 2, innerStrength: 0, color: per-ship, quality: 0.3` |
| Projectile glow | Projectile Container (all projectiles grouped) | pixi-filters GlowFilter | `distance: 8, outerStrength: 3, innerStrength: 0, color: per-weapon, quality: 0.3` |
| Global bloom | app.stage | pixi-filters AdvancedBloomFilter | `threshold: 0.4, bloomScale: 0.8, brightness: 1.1, blur: 4, quality: 4` |

**Performance rule:** Never apply GlowFilter to individual sprites. Always apply to a Container grouping all objects of the same type.

### Explosions

| Property | Value |
|----------|-------|
| Implementation | PixiJS Graphics circles expanding and fading |
| Small explosion (bullet kill) | Start radius 4px, expand to 16px over 300ms, fade opacity 1.0 to 0.0 |
| Large explosion (bomb kill) | Start radius 8px, expand to 32px over 500ms, fade opacity 1.0 to 0.0 |
| Color | Inner: `#ffffff`, outer: `#ffaa33`, edge: `#ff6600` (radial gradient effect via layered circles) |
| Particle count | 6-10 debris particles per explosion, random velocity, fade over 400ms |

### Engine Exhaust

| Property | Value |
|----------|-------|
| Implementation | PixiJS Graphics small circles behind ship |
| Trigger | Only while thrust input is active |
| Particle spawn rate | 1 per frame while thrusting |
| Particle size | 2-4px radius (random) |
| Particle color | `#00ff88` fading to `#004422` over lifetime |
| Particle lifetime | 200-400ms (random) |
| Particle velocity | Opposite of ship facing direction, with slight random spread |

## Audio Contract (UIEX-06)

| Sound | Trigger | Loop | Stereo Pan | Priority |
|-------|---------|------|------------|----------|
| thrust | Player thrusting (local only) | Yes (loop while held) | Center | Low (ambient) |
| bulletFire | Local player fires bullet | No | Center | High |
| bombFire | Local player fires bomb | No | Center | High |
| explosion | Any player death in viewport | No | Positional (left/right based on screen x) | High |
| bounce | Local ship hits wall | No | Center | Medium |
| death | Local player dies | No | Center | High |

**Audio initialization:** Create Howl instance on first user interaction in ship-select overlay. This satisfies browser autoplay policy.

**Volume:** Master volume 0.5 default. No per-category volume controls in v1 (defer to v2).

**Format:** WebM primary, MP3 fallback. Single audio sprite file containing all sounds.

## Interaction Contract

### Key Bindings

| Key | Action | Notes |
|-----|--------|-------|
| Arrow keys | Ship movement | Existing (unchanged) |
| Spacebar | Fire bullet | Existing (unchanged) |
| F | Fire bomb | Remapped from Tab (resolves Tab conflict from RESEARCH open question 1) |
| Tab | Toggle scoreboard | Show while held, hide on release |
| Enter | Open/send chat | Toggle chat input focus |
| Escape | Close chat | Cancel chat without sending |
| 1/2/3 | Ship select | Only active on ship-select overlay |

### State Transitions

```
[Ship Select] ---(pick ship)---> [Playing]
[Playing] ---(die)---> [Dead/Respawning] ---(server respawn)---> [Playing]
[Playing] ---(hold Tab)---> [Playing + Scoreboard visible]
[Playing] ---(press Enter)---> [Playing + Chat active (input captured)]
[Playing + Chat] ---(Enter/Escape)---> [Playing]
[Playing] ---(game over)---> [Game Over Screen] ---(5s timeout)---> [Ship Select]
```
